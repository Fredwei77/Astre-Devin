【测试方法】
1. 运行测试脚本:
   test-login-translation.bat

2. 手动测试:
   - 打开 http://localhost:3000/login.html
   - 切换到英文 → 验证所有文本是英文
   - 切换到中文 → 验证所有文本是中文

【验证要点】
✓ 页面标题: "Welcome Back to Destiny Realm" / "欢迎回到命运之境"
✓ 标签按钮: "Login" / "登入", "Register" / "注册"
✓ 表单字段: "Email Address" / "邮箱地址"
✓ 按钮文本: "Login" / "登入账户"
✓ 错误消息: "Please enter a valid email" / "请输入有效的邮箱地址"
✓ 密码强度: "Password Strength: Weak" / "密码强度：弱"
✓ 社交登录: "Login with Google" / "使用 Google 账户登入"

【关键文件】
- translations.js (添加登录翻译)
- login.html (添加 data-i18n 属性)
- login.js (使用翻译函数)

【翻译键示例】
login.title - 页面标题
login.email.label - 邮箱标签
login.email.placeholder - 邮箱占位符
login.button.login - 登录按钮
login.error.invalidEmail - 无效邮箱错误
register.passwordStrength.weak - 密码强度弱

【常见问题】
Q: 切换语言后没有更新？
A: 刷新页面 (Ctrl+F5) 或清除缓存

Q: 错误消息还是中文？
A: 检查 translations.js 和 unified-i18n.js 是否加载

Q: 占位符没有翻译？
A: 确认使用了 data-i18n-placeholder 属性

【控制台日志】
应该看到:
[Translations] Loaded into i18n system
[i18n] Language changed to: en
