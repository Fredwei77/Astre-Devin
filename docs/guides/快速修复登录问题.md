# 快速修复登录问题

## 🔴 问题

注册成功但登录失败，显示"Invalid login credentials"

## 💡 原因

用户注册后邮箱未验证，Supabase默认要求验证邮箱才能登录。

## ✅ 解决方案（3选1）

### 方案1: 禁用邮箱验证（最快）⭐

1. 访问Supabase Dashboard:
   ```
   https://supabase.com/dashboard/project/izkcgqvxecfxqtgxpmaj/auth/settings
   ```

2. 找到 **"Confirm email"** 选项

3. **关闭**这个开关

4. 点击 **Save**

5. 刷新登录页面，重新尝试登录

### 方案2: 手动验证用户

1. 访问用户管理:
   ```
   https://supabase.com/dashboard/project/izkcgqvxecfxqtgxpmaj/auth/users
   ```

2. 找到用户 `fredweir7@gmail.com`

3. 点击用户进入详情

4. 点击 **"Confirm Email"** 按钮

5. 返回登录页面，重新登录

### 方案3: 使用诊断工具

运行诊断工具查看详细信息：
```bash
debug-login-issue.bat
```

## 🧪 测试步骤

完成上述任一方案后：

1. 完全关闭浏览器

2. 重新打开登录页面

3. 输入邮箱: `fredweir7@gmail.com`

4. 输入密码

5. 点击登录

6. 应该能成功登录并跳转到主页

## 📋 快速检查清单

- [ ] 已访问Supabase Dashboard
- [ ] 已禁用邮箱验证 或 已手动验证用户
- [ ] 已保存设置
- [ ] 已关闭浏览器
- [ ] 已重新打开登录页面
- [ ] 已重新尝试登录

## 🔍 如果仍然失败

### 检查1: 密码是否正确

- 密码至少6个字符
- 确认没有输入错误

### 检查2: 用户是否存在

在Dashboard的Users中确认用户存在

### 检查3: 使用诊断工具

```bash
debug-login-issue.bat
```

在诊断工具中：
1. 检查连接状态
2. 测试注册新用户
3. 测试登录
4. 查看详细错误信息
