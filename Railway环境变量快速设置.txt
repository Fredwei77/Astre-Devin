╔══════════════════════════════════════════════════════════════════════╗
║                     🔧 Railway 环境变量快速设置                      ║
╚══════════════════════════════════════════════════════════════════════╝

📍 在Railway控制台 → Variables 标签页中逐个添加以下变量：

🌍 基础配置
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
变量名: NODE_ENV
变量值: production

变量名: PORT  
变量值: 3000

变量名: FRONTEND_URL
变量值: https://astredevin.netlify.app

🤖 OpenRouter AI 配置
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
变量名: OPENROUTER_API_KEY
变量值: sk-or-v1-你的OpenRouter密钥
📝 获取地址: https://openrouter.ai → API Keys

💳 Stripe 支付配置
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
变量名: STRIPE_SECRET_KEY
变量值: sk_test_你的Stripe秘钥

变量名: STRIPE_PUBLISHABLE_KEY  
变量值: pk_test_你的Stripe公钥

变量名: STRIPE_WEBHOOK_SECRET
变量值: whsec_你的Webhook密钥
📝 获取地址: https://dashboard.stripe.com → Developers → API keys

🗄️ Supabase 数据库配置
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
变量名: SUPABASE_URL
变量值: https://你的项目ID.supabase.co

变量名: SUPABASE_SERVICE_KEY
变量值: 你的Supabase服务角色密钥
📝 获取地址: https://supabase.com/dashboard → 项目 → Settings → API

🔐 JWT 认证配置  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
变量名: JWT_SECRET
变量值: 64位随机字符串
💡 生成命令: node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

╔══════════════════════════════════════════════════════════════════════╗
║                           📋 操作步骤                                ║
╚══════════════════════════════════════════════════════════════════════╝

1. 在Railway项目页面点击 "Variables" 标签
2. 点击 "New Variable" 按钮  
3. 输入变量名 (例如: NODE_ENV)
4. 输入变量值 (例如: production)
5. 点击 "Add" 保存
6. 重复步骤2-5，直到所有变量添加完成

╔══════════════════════════════════════════════════════════════════════╗
║                        ⚠️ 重要注意事项                               ║
╚══════════════════════════════════════════════════════════════════════╝

🔴 必须替换所有 "你的密钥" 为实际的密钥值
🟡 确保API密钥有效且未过期
🟢 添加完所有变量后，服务会自动重启并生效
🔵 可以随时修改变量值，修改后会自动重新部署

╔══════════════════════════════════════════════════════════════════════╗
║                          ✅ 验证部署                                 ║
╚══════════════════════════════════════════════════════════════════════╝

部署成功的标志:
- Railway项目状态显示为 "Active"
- 可以访问分配的域名 (例如: https://项目名.railway.app)
- 访问 /health 端点返回成功响应
- 日志中没有环境变量相关错误

如果遇到问题:
- 检查 "Deployments" 标签查看部署日志
- 确认所有环境变量都已正确设置
- 检查API密钥是否有效
